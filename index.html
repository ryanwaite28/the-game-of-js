How to play JS!

A guide on understanding JavaScript and Programming

There is a saying that goes:

You can't win a game you don't know how to play...


The purpose of this page is to teach the rules of JavaScript, and in some cases programming in general, so one can feel confident reading and writing code! Follow these rules and you will be coding with confidence like an expert! The best way to learn is practice and just doing it for a long time.

The console in the developer tools is a great place to play around with code. Google Chrome is recommended for learning web development.

One cannot be expected to know everything about programming but rather understand the foundation of programming, because all other aspects stems from the foundation. If you ever need to find an answer to how a language works, read its documentation. For JavaScript, the official documentation is here:
https://developer.mozilla.org/en-US/docs/Web/JavaScript

The rules below are in no particular order.



The Rules

  • Everything is an object

    Reference: Working with objects


    In object-oriented programming, an object is simply a data structure. Data structures are used for holding, organizing, and manipulating data/information. Essentially, objects have state and behavior. Objects have a collection of information associated with it, which are known as properties. When a property is a behavior(a.k.a function), it is called a method.

    The state is the current information an object holds.

    The behavior are the methods that an object can perform.

    The methods are often, if not all the time, defined to access/manipulate/modify the state of the object.

    Here is an analogy: a phone is an object; it has state and behavior. For example, some of the state a phone can have is it's color, model, carrier, size, brand, etc. Phones can also do things, like call and text.

  • Properties are accessed with dot-notation or bracket-notation

    Reference: Property access


    You can access an object's property by using the dot notation or the bracket notation. For Example: 

                        objectName.propertyName
                    // or
                    objectName['propertyName']

  • There are different types of objects and data

    Reference: Data Types


    Everything in JavaScript is an object, however there are different types of objects to represent different types of information. For example, letters are different than numbers and numbers are different than dates. Although there may be different types of objects, they all work the same due to being based on the same foundation; they have state and behavior.

    There are built-in object types, such as object literal, arrays, and dates. You can also define your own types of objects!

  • Objects are created from a Class

    Reference: Class


    In object-oriented programming, objects are created from a class. A class is a blueprint from with an object type is created from. For example, imagine a blueprint for a house. Multiple houses can be created using the same blueprint. The different houses may have different furniture(information) in them but they all have/share the same structure(class). A class can be created from another class, this is called "sub-class". That sub-class will inherit all of the properties from it's "parent-class".

    A class outlines the properties(state and behavior) that an object will have. An object is an instances of a class.

  • JavaScript is Prototype-Based

    Reference: Prototype-Based | Explanation


    In JavaScript, objects are not created from a class because JavaScript does not follow the class concept. Rather, in JavaScript, objects are created from something called a Prototype.

    In class based languages, objects cannot be extended after they are created. By extended, i mean they can only have the properties defined in its class; it cannot be assigned any more properties after being instantiated.

    In JavaScript, any object can have whatever properties a developer decides to give it; there is no required convention to follow and no requirement to have only what was defined in its prototype.

    Because of the way JavaScript behaves, it is often referred to as a loosely-typed/weak language.

  • There are primitives

    Reference: Primitive


    There are things in programming called primitives. A primitive is data that is not an object and has no state/behavior; they are incapable of being changed. For example, there is a primitive called Boolean. A boolean is like a switch, it can only be 1 of 2 values: yes or no, on or off. In programming, the two Boolean values are true or false. True will always be true and false will always be false. Numbers are primitives too: 1 will always be 1 and 2 will always be 2. There is nothing that can change that.

    This somewhat hinders the learning and understanding of objects in programming(JS in particular) because everything is considered an object, including numbers, but number are also considered a primitive, which cannot change. The reason JS also considers a number an object is because there is something specific to that language, called constructor function. Basically, there is a wrapper on top of primitives that allows them to work like objects, having state and behavior.

  • Variables store information

    Reference: Variables


    In programming, variables are use to hold a reference to data/information. Variables can store a reference to just about anything!

  • Variables are defined with a keyword

    Reference: Declaring variables


    Variables are defined using a variable keyword, then giving a name, then optionally assigning a value. Variables are created by using one of the following keywords: var, let, or const. For example: 

                        var name = 'Ryan';
                    let age;

  • Variables can be re-assigned a value

    Reference: Assignment


    After a variable has been created, it can have a diferent value at a different time. For example: 

                        var name = 'Ryan';

                    /// later ...

                    name = 'something else';

    Notice how there were no keywords involved with re-assigning a value. That is because the variable was already defined.

  • Functions are defined with a keyword

    Reference: Declaring functions


    Functions are defined using the function keyword, then giving a name, followed by parenthesis, then adding a pair of curly braces for the function body. For example: 

                        function example() {

                    }

  • User-Defined variables cannot have the same name as a keyword

    Reference: Keywords


    In programming, there are special words called keywords that are built into a language. They are reserved for certain functionalities of the language. That way, the language knows what to do when it sees it. For example, the var keyword is reserved for creating variables. If you were to create a variable with the name "var", the language would not know if you wanted to create a variabled or call the name of one.

  • Functions can accept information as input and return information as output

    Reference: Function arguments/return


    Functions are call "sub-programs" because they perform a specific task through a series of steps. 

                        function double_num(number) {
                      return number * 2;
                    }

    Functions don't have to return anything when being defined. In that case, JavaScript will return undefined by default. So in theory, all functions will return something.

  • Functions can accept multiple information but only return 1 value

    Reference: Function arguments/return


    Functions can accept many inputs, but only return 1 value. This is because the return keyword stops a function exection from continuing past where that return keyword was used. For Example 

                        function add(numberA, numberB) {
                      return numberA + numberB;
                    }

    Functions don't have to return anything when being defined. In that case, JavaScript will return undefined by default. So in theory, all functions will return something.

  • Functions don't run until they are called

    Reference: Function call

    In the above example, the double_num function was only defined, meaning it was created and designed on what it will do. It won't run until it gets called.

    To call a function, use its name followed by parenthesis. For example: 

                        double_num(2);

    Notice the value in the parenthesis, it is an actual value. When the function double_num was defined, it was defined by accepting an input, called number. That is just a reference to a value that gets inputted to the function when it gets called.

    The returned value can also be stored in a variable at the time of it being defined. For Example: 

                        var numberFour = double_num(2);

  • Functions can call other functions


    That is what makes programming, programming. Functions can call other functions. For example: 

                        function triple_num(num) {
                      return num * 3;
                    }

                    function square_trippled_num(number) {
                      var numberTrippled = triple_num(number);
                      return numberTrippled * numberTrippled;
                    }


                    square_trippled_num(4); // 144

    Notice how triple_num was being called within square_trippled_num.

  • You can chain/consolidate statements together

    Reference: Chaining


    Every time you call a variable, you get the value that it holds. Every time you call a function, you get what was returned. If you know the object being return, you can chain on a property or method call! jQuery does this heavily: For example: 

                        $('body').find('p').css('color', 'red');

  • Code compilation

    Reference: Code compilation


    Code is compiled and executed in a sequence. Code that doesn't execute in sequence causes an error. The following would give a ReferenceError: 

                        console.log(name); // name does not exist yet
                    let name = 'ryan';

  • Declaration vs. Expression

    Reference: Declaration vs. Expression


    Function declarations(functions created using the function keyword) and variables are hoisted. Variables that are hoisted do not hoist their value, just the name/identifier. See the reference link for more info.

    In computer programming, a declaration is a language construct that specifies properties of an identifier: it declares what a word (identifier) "means".

    Wikipedia - Declaration

    An expression in a programming language is a combination of one or more explicit values, constants, variables, operators, and functions that the programming language interprets (according to its particular rules of precedence and of association) and computes to produce ("to return", in a stateful environment) another value. This process, as for mathematical expressions, is called evaluation.

    Wikipedia - Expression

  • Synchronous vs Asynchronous

    Reference: Sync vs Async


    In programming, things happen in order and you wait for it to finish before moving on to another task; that is called synchronous. Tasks can happen outside of an order, that is called asynchronous. One cannot "wait" for an asynchronous because the order would have continued on when an asynchronous code is executed.
